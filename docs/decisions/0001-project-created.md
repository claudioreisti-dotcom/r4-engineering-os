# Decision 0001: Create R4 Engineering OS

## Status

Accepted

## Context

A R4 precisava reduzir dependência de ferramentas específicas de IA no desenvolvimento de software.

Durante a análise, ficou claro que o problema real era maior: conhecimento estratégico estava disperso entre chats, prompts, ferramentas e documentos específicos de projetos.

## Decision

Criar o R4 Engineering OS como produto independente, reutilizável por todos os projetos da R4.

## Rationale

O R4 EOS deve permitir que projetos como DOK1, iVet e IAReis sejam governados por um modelo comum de engenharia AI-Driven e Human-Governed.

## Consequences

- O DOK1 passa a ser projeto de referência, não origem exclusiva.
- O R4 EOS terá repositório próprio.
- Ferramentas serão adapters.
- Conhecimento será versionado.
